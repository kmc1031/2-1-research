# 3D DT-CWT 기반 비디오 전처리 연구 개선 사항 및 향후 과제

본 문서는 현재 구현된 3D DT-CWT 비디오 전처리 파이프라인의 한계점을 분석하고, 연구의 논리적 타당성 및 시스템 성능을 강화하기 위한 개선 과제를 정리한 것입니다.

## 1. 알고리즘 및 화질 측면의 개선

* **색상(Chroma) 노이즈 처리 도입**
    * **현황:** 연산 효율성을 위해 Y(휘도) 채널에만 전처리를 적용하고, U/V 채널은 원본 데이터를 그대로 유지하고 있음.
    * **문제점:** 색상 노이즈가 심한 저조도 영상 등에서는 전체적인 시각적 품질 향상에 한계가 발생함.
    * **개선안:** U/V 채널에 대해 간소화된 2D/3D 웨이블릿 변환을 적용하거나 해상도를 낮춰 처리하는 방식의 크로마 노이즈 제거 로직 추가.
* **실행 환경 간 임계값(Threshold) 처리 로직 통일**
    * **현황:** GPU(`dtcwt_cuda.py`) 환경에서는 노이즈 및 서브밴드 분산을 고려한 '시공간 적응형 임계값(BayesShrink 기반)'을 사용하나, CPU(`dtcwt_processor.py`) 환경에서는 단순 고정 임계값 감산(Soft Shrinkage)을 사용함.
    * **문제점:** 하드웨어 환경에 따라 출력 결과물의 품질 파편화 발생.
    * **개선안:** CPU 처리 경로에도 GPU와 동일한 수준의 적응형 임계값 산출 로직 이식.
* **장면 전환(Scene Change) 대응 로직 추가**
    * **문제점:** 청크(Chunk) 단위 처리 중 장면 컷이 바뀌면 이전/이후 프레임이 시간축(T)으로 섞여 잔상(Temporal Smearing) 아티팩트가 발생할 위험이 있음.
    * **개선안:** FFmpeg의 장면 전환 감지 데이터를 파이프라인으로 가져와, 컷이 바뀌는 구간에서는 시간축 필터링을 끊어주거나 초기화하는 로직 구현.

## 2. 성능 및 연산 효율성 최적화

* **오버랩(Overlap) 연산 중복 최소화**
    * **현황:** 시간축 경계 아티팩트 방지를 위해 Overlap-Save 방식을 사용하며, 청크마다 지정된 개수(예: 4프레임)를 중복해서 처리함.
    * **문제점:** 전체 처리 프레임의 상당 부분(기본 설정 기준 약 33%)이 무거운 3D DT-CWT 연산을 중복으로 수행하여 컴퓨팅 자원 낭비.
    * **개선안:** 이전 청크에서 계산된 웨이블릿 계수나 중간 버퍼 상태를 캐싱하여 재사용하는 방식 연구.
* **CPU-GPU 간 데이터 병목 현상 완화**
    * **현황:** FFmpeg 파이프(CPU) → PyTorch 텐서(GPU) → NumPy 변환(CPU) → x264 인코더 파이프(CPU) 구조로 메모리 복사 비용이 큼.
    * **개선안:** NVDEC/NVENC 기반 하드웨어 가속 파이프라인 도입을 통해 비디오 디코딩-전처리-인코딩 과정을 GPU 메모리 내에서 직접 수행.

## 3. 연구 논리 및 타당성 강화를 위한 실험 설계 (Ablation Study)

화질 향상이 기타 부가 기술이 아닌 3D DT-CWT 알고리즘 본연의 우수성에서 기인했음을 증명하기 위한 비교 실험군 추가가 필요함.

* **변환(Transform) 대조군(Baseline) 평가 도입**
    * **실험 1:** 원본 영상 vs 제안 모델 (효과 입증)
    * **실험 2:** 단순 2D 공간 필터(Gaussian/Bilateral Blur 등) vs 제안 모델
    * **실험 3:** 일반 3D 이산 웨이블릿 변환(DWT) vs 제안 모델 (DT-CWT의 이동 불변성 및 방향성 효과 증명)
* **절제 연구(Ablation Study) 수행**
    * GPU의 '시공간 적응형 임계값' 기능을 끈 상태(고정 임계값)의 RD Curve 비교.
    * 오버랩(Overlap) 적용 유무에 따른 경계면 화질 손실(EPSNR 등) 수치화.
* **잔차 이미지(Residual Image) 시각적 분석 추가**
    * (원본 영상) - (전처리된 영상)의 차이 값을 시각화.
    * 추출된 잔차에 구조적 디테일(Edge)이 포함되지 않고 무작위 노이즈만 제거되었음을 증명.

## 4. 소프트웨어 구조 및 유연성

* **파라미터 하드코딩 제거 및 CLI 도구화**
    * **현황:** 파일 경로, 비트레이트(`100k`), Chunk Size, 인코딩 Preset 등이 스크립트 내부에 고정되어 있음.
    * **개선안:** `argparse` 등을 사용하여 대규모 데이터셋(다양한 해상도, 여러 비트레이트 타겟) 자동화 실험이 가능하도록 구조 개선.
* **홀수 크기 해상도 패딩 알고리즘 정교화**
    * **현황:** 홀수 크기의 차원이 입력되면 `mode='edge'` 옵션으로 단순히 패딩 후 잘라냄.
    * **문제점:** 영상 가장자리의 주파수 왜곡 가능성.
    * **개선안:** 웨이블릿 변환에 더 적합한 대칭 확장(Symmetric Extension) 기법 적용을 통한 경계선 아티팩트 최소화.
